1. AgentCore Runtime

The secure serverless execution engine for AI agents — built on Firecracker microVMs, supporting 8-hour sessions, session isolation, multi-protocol, and any AI framework without infrastructure management.

Firecracker microVMs8h sessionsFramework agnostic
💡 Analogy

Runtime is like a premium hotel where every guest gets their own private suite — completely isolated, fully furnished from scratch. When the guest checks out, the suite is demolished and rebuilt new. The hotel handles all plumbing, electricity, and maintenance. You only pay for occupied time. Each suite lasts up to 8 hours and supports any conversation, no matter how complex.

Core Capabilities
  • Firecracker microVM per session — true security isolation
  • Sessions up to 8 hours — longest in the industry
  • Multi-protocol: MCP, A2A, AGUI, HTTP/REST, WebRTC
  • Session affinity via runtimeSessionId
  • Auto-scaling — serverless, pay per compute second
  • Works with LangGraph, CrewAI, Strands, OpenAI SDK, ADK
  • Any model: Claude, Nova, GPT-4, Gemini, Llama, Mistral
  • Bidirectional streaming for voice agents (Dec 2025)
  • Spring AI SDK support (GA April 14, 2026)
  • VPC/PrivateLink for enterprise network isolation

# Multi-turn: same runtimeSessionId = shared context

2. AgentCore Memory

Fully managed short-term and long-term memory for AI agents. 5 extraction strategies including Episodic Memory (Dec 2025). Streaming notifications for long-term memory updates (March 2026). No vector DB to manage.

Short + Long-termVector-backed5 strategies
💡 Analogy

Memory is like a personal assistant with two notebooks plus a case history binder. Notebook 1 (short-term) = today's desk notes. Notebook 2 (long-term) = indexed filing cabinet of key facts, preferences, and past decisions. The binder (episodic) = complete patient records: what was tried, what worked, what failed — enabling learning across thousands of cases. Plus (March 2026): the assistant now calls you the moment a new fact is filed.

3 Key Identifiers

When the agent says "create a Jira ticket", the semantic concierge finds the right tool instantly among thousands, without the agent needing to know every tool by name.

4. AgentCore Identity

Purpose-built identity and credential management for AI agents — OAuth 2.0 flows, workload identities, secure token vaults, and user-delegated access. Works standalone outside AgentCore Runtime (EKS, Lambda).

OAuth 2.0Workload IdentityToken Vault
💡 Analogy

Identity is like a corporate security office + notary system for AI agents. When you send your AI agent (intern) to do work on your behalf, Identity gives it a temporary, scoped access badge: "Authorized for User Sarah — only Jira tickets — expires at 5 PM." The agent never sees your master password; it gets a limited, fully audited token. If compromised, the badge expires worthlessly.

OAuth Flow — 4 Phases
1

User Authentication

User authenticates via Cognito, Entra ID, or Okta — establishing identity context and consent scope for agent.

2

Workload Identity Issued

A unique ARN-based workload identity is created for the agent — every Runtime and Gateway deployment gets one automatically.

3

Token Vault Creates Scoped Token

AgentCore combines user token + agent identity into a scoped, time-limited opaque token. Raw credentials never leave the vault.

4

Secure Audited Tool Access

Agent calls Salesforce, GitHub, Slack with token — full CloudTrail audit of every credential usage, automatic expiry.

Identity Types
  • AgentCore::OAuthUser — OAuth JWT sub claim
  • AgentCore::IamEntity — Full IAM ARN
  • Workload Identity — Agent ARN in Identity Directory
Supported Identity Providers
🔵 Amazon Cognito 🔶 Microsoft Entra ID 🔸 Okta Auth0
OAuth Patterns
  • 3-Legged OAuth — Agent acts on behalf of a user
  • 2-Legged OAuth (M2M) — Agent acts as itself autonomously
  • SigV4 signing for AWS native resources
  • Custom claims for multi-tenant environments (Dec 2025)
  • Works standalone on EKS, Lambda, any compute

6. AgentCore Code Interpreter

A fully managed, isolated sandbox for AI agents to write and execute code — solving complex math, data analysis, visualizations, and multi-step workflows that pure language model reasoning cannot reliably handle.

Python, JS, TypeScript8h execution5GB S3 files
💡 Analogy

Code Interpreter is like a secure, isolated chemistry lab for your AI agent. The agent has all the chemistry knowledge (LLM reasoning), but for actual experiments (real computation), it needs a physical lab. The lab is completely isolated — nothing inside contaminates anything outside. All instruments are pre-installed (popular libraries preloaded), every experiment is fully logged, and precise results are returned to the agent.

Capabilities
  • Execute Python, JavaScript, TypeScript in isolated sandbox
  • Process large datasets up to 5GB via S3 references
  • Generate data visualizations — charts returned as files
  • Complex math, statistics, scientific computing
  • File format transformation and conversion
  • Multi-step iterative workflows — stateful execution
  • Default 15-min, extendable up to 8 hours
  • Pre-installed: pandas, numpy, matplotlib, scikit-learn
AGENT-GENERATED CODE (Runs in isolated sandbox)
import pandas as pd, matplotlib.pyplot as plt

# Load dataset from S3 (up to 5GB)

# Multi-step analysis

# Auto-generate chart

8. AgentCore Policy (Cedar)

Real-time fine-grained access control for every tool call using Cedar — the same open-source policy language powering Amazon Verified Permissions. Decouples authorization from agent code. Added at re:Invent Dec 2025.

Cedar PAASReal-time enforcementDec 2025
💡 Analogy

Policy is like a real-time bank transaction approval system. Every time your AI agent requests a tool action, Cedar runs a check in <1ms: "Is agent A allowed to call tool X for user Y right now?" — just like your bank instantly checks "Is this card authorized for this merchant?" The decision is enforced before execution, not logged after. And it can't be bypassed with a clever prompt.

How Cedar Policy Works
1

Agent Requests Tool Action

Agent in Gateway or Runtime attempts to call a specific tool with specific parameters.

2

Cedar Engine Evaluates in <1ms

Checks: principal (who is the agent?), action (what tool?), resource (what data?), context (time, conditions).

3

ALLOW or DENY — Before Execution

Policy decision is enforced externally from the LLM — cannot be prompt-injected or bypassed. Full audit trail.

CEDAR POLICY EXAMPLE
// Only allow support-agent to read CRM
// for premium-tier customers during business hours
permit (
  principal == AgentCore::Runtime::"support-agent",
  action   == Gateway::Action::"getCustomerRecord",
  resource  == Gateway::Tool::"crm-lookup"
) when {
  context.user.tier == "premium" &&
  context.hour >= 9 && context.hour <= 17
};
Key Advantages
  • Policies stored externally — update without redeploying agents
  • Integrates with AgentCore workload identities
  • Prevents prompt injection attacks on tool access
  • Every decision logged to CloudTrail for audit
  • Natural language policy authoring option also available

9. Protocols: MCP, A2A, AGUI & More

AgentCore supports all major open agentic protocols — enabling agents to communicate with tools (MCP), each other (A2A), and human UIs (AGUI) via standardized, interoperable standards.

🔌
MCP
Model Context Protocol — Anthropic open standard. Agent ↔ Tools. Gateway IS a full MCP server.
🤖
A2A
Agent-to-Agent — Google open standard. AgentCard capability discovery. Task delegation between agents.
💬
AGUI
Agent-User Interaction — CopilotKit standard. Real-time bidirectional UI streams for frontend apps.
🌐
WebRTC/WS
Real-time streaming — Bidirectional audio for voice agents. Added Dec 2025.
💡 Protocol Analogy

Think of protocols as agent communication languages: MCP is English for agent-tool conversations, A2A is the inter-office memo standard between departments, AGUI is the presentation layer showing work to humans, and WebRTC/WS is the telephone line for live voice. Every AgentCore agent speaks all of these fluently — simultaneously.

10. Bidirectional Streaming

Real-time bidirectional communications for voice AI agents, live assistants, and concurrent multi-channel I/O — via WebSocket and WebRTC. The breakthrough capability from AWS re:Invent December 2025.

WebSocket + WebRTCVoice AI enabledre:Invent Dec 2025
💡 Analogy

Bidirectional Streaming upgrades your AI agent from a letter-writer to a phone caller. Before: write a letter (send request) → wait for reply (polling). Now: real-time phone call — agent hears you immediately, thinks while listening, responds as it reasons, can be interrupted naturally. This is what enables truly conversational, voice-based AI agents that feel like talking to a human.

Use Cases
  • Voice AI agents — speak naturally, get instant voice responses
  • Real-time code completion and explanation
  • Live customer support with instant agent responses
  • Interactive data analysis with live commentary
  • Multi-modal: simultaneous audio + data streams
  • Real-time translation and interpretation agents
PYTHON — BIDIRECTIONAL VOICE STREAM
import asyncio

async def voice_agent():
  async with client.invoke_bidi_stream(
    agentRuntimeArn="arn:aws:bedrock-agentcore:...",
    protocol="WebRTC"
  ) as stream:
    # Send audio chunks as they arrive from mic
    async for chunk in mic_stream():
      await stream.send(chunk)
      # Receive response in real time
    async for resp in stream.receive():
      await play_audio(resp)

Quick Start — 5 Steps to First Agent

Launch your first AgentCore agent in under 10 minutes using Strands SDK. Requires AWS account + Python 3.9+.

⚡

0 → Deployed Agent in 5 Steps

1

Install Dependencies

SHELL
pip install strands-agents boto3
aws configure  # Set region: us-east-1 or us-west-2
2

Create agent.py

PYTHON
from strands import Agent
from strands.tools import calculator

agent = Agent(
  model="us.amazon.nova-pro-v1:0",
  tools=[calculator],
  system_prompt="You are a helpful financial advisor."
)
result = agent("What is compound interest on $10,000 at 7% for 10 years?")
print(result)
3

Test Locally

SHELL
python agent.py
# Compound interest = $19,671.51 (96.7% growth over 10 years)
4

Deploy to AgentCore Runtime

CLI
agentcore deploy --name financial-advisor --region us-east-1
# ✓ arn:aws:bedrock-agentcore:us-east-1:123:agent-runtime/fin...
5

Invoke from Any Application

PYTHON — BOTO3
import boto3, json
client = boto3.client("bedrock-agentcore", region_name="us-east-1")
response = client.invoke_agent_runtime(
  agentRuntimeArn="arn:aws:bedrock-agentcore:us-east-1:...",
  qualifier="DEFAULT",
  payload=json.dumps({"prompt": "Analyze my Q4 2025 budget"})
)
